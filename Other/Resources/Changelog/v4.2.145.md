# Release notes

## Changes

- PlantUML-diagrams are now rendered with a pinned Eclipse-Temurin-JRE (downloaded into the global cache via the new download_jre/ensure_jre_is_available) instead of the host's java; together with the bundled DejaVu-font this makes the generated diagram-SVGs byte-identical across machines (e.g. a Windows-client and the Debian-build-container)
- the JRE-version can be pinned per repository in '<repo>/Other/Resources/Dependencies/JRE/Version.txt' (analogous to PlantUML and OpenAPIGenerator), with a fallback to the default version bundled with ScriptCollection
- nodejs-linting now normalizes the OpenAPI-generator's '.openapi-generator/FILES' metadata-file to LF so it does not drift between build-hosts
- documented the pinned JRE and the diagram-determinism-rationale in DownloadableTools.md
- added the sccreateskill command (create_skill in ScriptCollectionCore) which generates a skill-folder ('skills/<name>/' with a lightweight skill.json and a lazy-loaded detail.md) in the given repository, the current git-repository or the user-folder
- scprintfilecontent now supports the optional --fromline/--toline parameters to restrict the printed output to a range of lines
- the common-tasks now propagate the repository-level dependency-version-pins (e.g. PlantUML, OpenAPIGenerator, JRE) into each codeunit
