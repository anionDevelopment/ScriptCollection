# Release notes

## Changes

- added the function normalize_line_endings (also available via the new CLI-command scnormalizelineendings) which normalizes all physical line-endings of a file to LF
- added the helper get_not_git_ignored_files_of_folder which lists the not-git-ignored files of a folder (optionally filtered by extension)
- the generated OpenAPI-specification-files (.json and .yaml) now always use LF line-endings (also on Windows); carriage-returns inside JSON string-values are removed as well
- dotnet-linting now normalizes the line-endings of all not-git-ignored *.cs-files of the codeunit and its test-project to LF
- nodejs-linting now normalizes the line-endings of all not-git-ignored *.ts-, *.js-, *.html-, *.css-, *.scss- and *.sass-files of the codeunit to LF
