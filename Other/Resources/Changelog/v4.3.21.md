# Release notes

## Changes

- fixed the calculated semver-version when the current commit is on a tag but the repository has uncommitted changes (the next version is now computed by gitversion itself instead of returning the tag-version); the result is cached in-memory and persistently below `<repository>/.ScriptCollection/Cache`
- added the CLI-command `sccheckpythonast` which checks python-files (single file or a folder recursively) for syntax-errors via the ast-module
- `scbuildcodeunits` now ensures `<repository>/.ScriptCollection/.gitignore` exists and ignores the cache-folder
